What is VisitPedia?

Introduction

Visitpedia is a tool that helps to identify the trending events on Google and Wikipedia from the visit counts of the keywords. These keywords when clustered together gives a bigger picture of the trending events on these websites.The evolution pattern of these events is concerned with changes in the content of the event over time and understanding how are these related. The Visitpedia tool works in following steps:
a)Data Pre-Processing: This is concerned with the structuring of data for visualization and analysis. The dataset here comprises of the visit counts indicating the number of visits of various events on Wikipedia per month. Those entries with very small event counts are discarded and the ones with higher values are kept.
b)Entry Event Analysis: After the data is prepared, analysis of the events is performed using an event detection algorithm. The visit count of the entries is visualized using charts that displays the sum of visit count of the events.
c)Critical Event extraction: The results extracted from the event analysis helps in categorising events as sudden(duration aware) or continuous events.
d)Event Analysis :The tool solves this time clustering problem to find events similar to the critical event that was extracted in the previous step. The results are split into subsets and these subsets might be merged in different combination to understand the underlying patterns that exist between the subsets.
e)Visualization and Interaction: The last step involves communicating the results of our analysis using some visualisation and interaction technique. The Event overview calendar, Critical Event River , Evolution graph and Text view and line graph are the visualisation techniques that can be used to represent the results of the different steps in the process.

Proposed Extension:

With this project, we have tried to create a tool that effectively evaluates the current events on Google and Wikipedia for a particular time frame based on the selection of the user.
